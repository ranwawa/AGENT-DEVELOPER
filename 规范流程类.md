# 小程序体验版本验证正确,但合并master后推送新的体验版却白屏	
## 问题描述	
在dev分支修改后自动推送	
github actions会自动部署体验版小程序	
验证ok后合并到master	
master 上的action自动重新部署体验版小程序	
完了提审睡觉	
第二在早上起来发现首页是白屏	
## 原因分析	
应该是在合并过程中代码冲突导致的逻辑出错	
## 问题解决	
新增核心流程的端到端验证	
在合并master之前一定要全跑一次,通过后才能合并

# 已经在小程序上测试过的功能,最后发生产的时候却丢失了 	
## 问题描述	
通过OMO计划模式新增的功能，在小程序上已经人工验证过了，后面不知道 怎么搞的又没了	
## 原因分析	
- worktree	
	1. OMO的计划模式会单独起一个worktree来工作	
	2. 不知道后面有没有合并到dev分支	
	3. 可能在跑其他任务的时候又把这个给搞掉了	
- 未commit	
	1. 有些任务完成之后直接推到了小程序体验版本	
	2. 可能没有自动commit和push，导致代码丢失	

## 问题解决	
- worktree	
	1. 先要搞懂omo计划模式的完整逻辑	
	2. 搞懂之前人为多注意合并dev     ok	
- 未commit	
	* 在agents.md中强制要求commit,push     ok	
	* 删除从npm命令中直接deploy的逻辑，避免ai搞混   ok
